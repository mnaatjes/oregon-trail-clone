# ADR 008: Adoption of Diátaxis Documentation Framework

## Context
The Oregon Trail Clone project requires a robust documentation strategy to manage increasing complexity. We select the **Diátaxis** framework to categorize content based on user needs.

## Decision

### 1. The Four Quadrants
* **Tutorials:** Learning-oriented.
* **How-to Guides:** Task-oriented.
* **Explanation:** Understanding-oriented (where ADRs and TDDs live).
* **Reference:** Information-oriented (API specs, glossaries).

### 2. Physical Structure
Documentation is mapped to these quadrants in the `docs/` directory to ensure predictable discovery.
